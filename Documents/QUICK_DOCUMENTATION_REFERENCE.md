# Quick Documentation Reference

A one-page quick reference for documenting backend changes.

---

## 🎯 Where to Document What

### Backend Changes

| What Changed | Document In |
|-------------|-------------|
| API Route | `BACKEND_API_ROUTES_DOCUMENTATION.md` |
| Module/Agent | `BACKEND_MODULES_DOCUMENTATION.md` |
| Database Schema | `BACKEND_DOCUMENTATION.md` |
| Configuration | `BACKEND_DOCUMENTATION.md` |
| Bug Fix | Relevant file + note in main doc |
| New Feature | All relevant files |

---

## 📝 Template Location

- **Backend Templates**: `BACKEND_DOCUMENTATION_TEMPLATE.md` (in this folder)

---

## ✅ Pre-Commit Checklist

- [ ] Code is complete and tested
- [ ] Template is filled out
- [ ] Documentation file is updated
- [ ] "Last Updated" date is set
- [ ] Code examples included
- [ ] Related docs cross-referenced

---

## 🚀 Quick Process

1. Make code changes
2. Copy relevant template from `BACKEND_DOCUMENTATION_TEMPLATE.md`
3. Fill out template
4. Add to documentation file
5. Update "Last Updated" date
6. Commit together

---

**Remember**: No code changes without documentation! 📚
